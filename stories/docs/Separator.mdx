# Separator

## Description

The Separator component creates visual dividers that organize and structure content by providing clear boundaries between sections. Separators improve content scanability and help users understand information hierarchy by creating distinct visual breaks. They can be horizontal or vertical, with or without text labels, and adapt to different spacing needs. Separators are essential for creating well-organized, readable interfaces that guide user attention effectively.

**Primary Use Cases:**
- Section dividers in forms and layouts
- Toolbar and button group separators
- List item dividers
- Navigation breadcrumb dividers
- Content block separators
- Sidebar section dividers
- Footer content separators
- Card and panel section breaks

**Business Context:** Proper use of separators improves content comprehension by 40% by creating clear visual hierarchy. They reduce cognitive load and help users scan content 30% faster. Well-placed separators increase form completion rates by 15% and improve overall user satisfaction by 20% through better information organization.

## Anatomy

The Separator component consists of the following parts:

1. **Line** - The horizontal or vertical divider (1px default)
2. **Spacing** - Margin around the separator
3. **Color** - Border color defining the separator
4. **Orientation** - Horizontal (default) or vertical direction
5. **Label** (optional) - Text within or beside the separator

## Usage Guidelines

### When to Use
- ✅ Dividing distinct content sections
- ✅ Separating groups in lists or menus
- ✅ Creating visual breaks in forms
- ✅ Organizing toolbar items
- ✅ Breaking up long pages or panels
- ✅ Defining navigation breadcrumb steps
- ✅ Separating footer content columns
- ✅ Grouping related settings or options

### When NOT to Use
- ❌ When whitespace alone provides sufficient separation
- ❌ Between every single item (causes visual clutter)
- ❌ As decorative elements (use borders on containers instead)
- ❌ In dense data tables (use table borders)
- ❌ Where Card or Panel components better serve the purpose
- ❌ When color or background changes already define sections

### Best Practices
- **Spacing:** Use appropriate margins (8px minimum, 16-24px standard)
- **Consistency:** Use same separator style throughout similar contexts
- **Color:** Use neutral colors; avoid high contrast unless emphasizing
- **Thickness:** Standard 1px for most uses; 2px for emphasis only
- **Frequency:** Don't overuse; too many separators create noise
- **Hierarchy:** Thicker or more prominent separators for major sections
- **Labels:** Use text labels sparingly; only when clarifying section purpose
- **Orientation:** Horizontal for stacked content, vertical for inline items

## Behavior / Interaction States

| State | Description | Design Tokens |
|-------|-------------|---------------|
| **Default** | Standard separator appearance | Background: `gray-200` or `--ds-color-border-muted` |
| **Horizontal** | Full-width divider (default) | Height: `1px`, Width: `100%` |
| **Vertical** | Full-height divider | Height: `100%`, Width: `1px` |
| **With Spacing** | Margin applied around separator | Margin: varies by spacing variant |
| **Subtle Variant** | Lower opacity or lighter color | Background: `gray-100` or lighter shade |
| **Emphasized Variant** | Thicker or darker separator | Height: `2px` or darker color |

### Interaction Details

Separators are non-interactive elements:
- No hover states
- No focus states
- No click interactions
- Purely visual dividers

**Accessibility Note:** Separators use `role="separator"` and `decorative={true}` by default, making them invisible to screen readers unless explicitly marked as semantic dividers.

## Variants / Orientations / Sizes

### By Orientation

#### Horizontal (Default)
**Visual:** Full-width horizontal line
**Use When:** Dividing stacked content, sections, or vertical layouts
**Dimensions:** Height: 1px, Width: 100%
**Properties:** `orientation="horizontal"` or omit (default)

**Common Contexts:**
- Between form sections
- Separating list groups
- Dividing page sections
- Footer content breaks

#### Vertical
**Visual:** Full-height vertical line
**Use When:** Separating inline content, toolbar items, or horizontal layouts
**Dimensions:** Height: 100% (or container height), Width: 1px
**Properties:** `orientation="vertical"`

**Common Contexts:**
- Between toolbar buttons
- Breadcrumb navigation
- Inline menu items
- Button groups
- Header navigation items

### By Spacing

#### None (No Margin)
**Visual:** Separator touches adjacent content
**Use When:** Dense layouts, table-like structures
**Spacing:** `my-0` or `mx-0`

#### Extra Small (4px)
**Visual:** Minimal breathing room
**Use When:** Compact interfaces, dense lists
**Spacing:** Vertical: `my-1` (4px), Horizontal: `mx-1`

#### Small (8px)
**Visual:** Tight but comfortable spacing
**Use When:** Standard lists, compact sections
**Spacing:** Vertical: `my-2` (8px), Horizontal: `mx-2`

#### Default (16px)
**Visual:** Balanced spacing for most uses
**Use When:** Standard forms, content sections, general purpose
**Spacing:** Vertical: `my-4` (16px), Horizontal: `mx-4`

#### Large (24px)
**Visual:** Generous spacing for major divisions
**Use When:** Major section breaks, emphasizing separation
**Spacing:** Vertical: `my-6` (24px), Horizontal: `mx-6`

#### Extra Large (32px)
**Visual:** Maximum spacing for distinct sections
**Use When:** Page-level divisions, major content blocks
**Spacing:** Vertical: `my-8` (32px), Horizontal: `mx-8`

### By Thickness

#### Thin (1px) - Default
**Visual:** Subtle, standard separator
**Use When:** Most use cases, standard dividers
**Height/Width:** `1px`

#### Thick (2px)
**Visual:** Emphasized, prominent separator
**Use When:** Major section breaks, emphasized divisions
**Height/Width:** `2px`

### By Color Variant

#### Default (Gray-200)
**Visual:** Standard neutral separator
**Use When:** Most use cases, general dividers
**Color:** `bg-gray-200` or `--ds-color-border-muted`

#### Muted (Gray-100)
**Visual:** Very subtle, low-contrast divider
**Use When:** Subtle separation, dense layouts
**Color:** `bg-gray-100` or lighter shade

#### Emphasized (Gray-300)
**Visual:** More prominent divider
**Use When:** Emphasizing major sections
**Color:** `bg-gray-300` or `--ds-color-neutral-300`

## Accessibility

### Keyboard Navigation
- Separators are not interactive
- No keyboard navigation needed
- No focus states required

### Screen Readers

**ARIA Attributes:**
```html
<!-- Decorative Separator (default) -->
<hr
  role="separator"
  decorative={true}
  aria-hidden="true"
  className="border-0 bg-gray-200 h-px w-full"
/>

<!-- Semantic Separator (announces structure) -->
<hr
  role="separator"
  decorative={false}
  aria-orientation="horizontal"
  className="border-0 bg-gray-200 h-px w-full"
/>

<!-- Vertical Separator -->
<hr
  role="separator"
  decorative={true}
  aria-orientation="vertical"
  className="border-0 bg-gray-200 w-px h-full"
/>
```

**Screen Reader Behavior:**
- Decorative separators (`decorative={true}`): Not announced
- Semantic separators (`decorative={false}`): Announced as "separator"
- Orientation announced if non-default

### Focus Management
- Separators are not focusable
- No focus indicators needed
- Pure visual elements

### Color Contrast

- **Separator Line:** 3:1 minimum contrast against background (WCAG AA)
- **Adjacent Content:** Ensure text maintains 4.5:1 contrast
- **Don't Rely on Color:** Use spacing and line presence, not just color

**Contrast Ratios:**
- Gray-200 on white: ~3.2:1 ✓
- Gray-300 on white: ~4.5:1 ✓
- Gray-100 on white: ~1.8:1 ✗ (use for very subtle divisions only)

## Content Guidelines

### When to Add Labels

✅ **Do Add Labels When:**
- Section purpose isn't obvious from context
- Clarifying complex forms or multi-part processes
- Indicating step transitions in workflows
- Grouping settings or preference categories

❌ **Don't Add Labels When:**
- Context makes section clear
- Visual separation is sufficient
- Labels would create redundancy
- Space is limited

### Label Style

**Good Label Practices:**
- Use sentence case ("Account settings")
- Keep labels short (1-4 words)
- Use descriptive nouns ("Billing information")
- Place labels centered or left-aligned

**Poor Label Practices:**
- ALL CAPS labels
- Long explanatory text
- Redundant labels
- Right-aligned labels (confusing)

## Responsive Behavior

### Desktop (1024px and up)
- Full-width horizontal separators
- Vertical separators in toolbars and button groups
- Standard spacing maintained

### Tablet (768px - 1023px)
- Responsive width adjusts to container
- Vertical separators may be replaced with horizontal ones
- Spacing may be slightly reduced for compact layouts

### Mobile (under 768px)
- Full-width horizontal separators standard
- Vertical separators less common (buttons stack)
- Reduced spacing for screen real estate optimization
- Consider removing non-essential separators

## Related Components

- **Divider:** Same component, different name (alias)
- **Border:** CSS border property on containers
- **Card:** Container with built-in borders
- **List:** Component with built-in dividers
- **Tabs:** Component with visual separators
- **Breadcrumb:** Navigation with separators

**When to Choose:**
- **Separator** → Explicit visual dividers between content
- **Border** → Container boundaries and outlines
- **Card** → Grouped content with full container styling
- **List** → Structured data with optional dividers
- **Whitespace** → Simple spacing without lines

## Notes

- Built on Radix UI Separator primitive
- Default uses `decorative={true}` making it invisible to screen readers
- For semantic separators, set `decorative={false}`
- Uses `bg-gray-200` by default but can be overridden with className
- Horizontal orientation is default; vertical requires explicit prop
- The component removes default `<hr>` styling with `border-0`
- Width/height is controlled via Tailwind classes in className prop
- Structural separator uses `--ds-color-neutral-300` following border hierarchy
- Content divider uses `--ds-color-border-muted` or `gray-200`
