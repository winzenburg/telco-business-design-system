# List

## Description

The List component displays structured content in a vertical layout, presenting items in a clear, scannable format. Lists organize related information with consistent styling, supporting various content types from simple text to complex multi-line items with icons, avatars, and actions. They are essential for displaying collections of data, navigation options, settings, and any content that benefits from ordered presentation.

**Primary Use Cases:**
- Service and product listings
- Settings and preference menus
- Contact and user directories
- File and document browsers
- Navigation sidebars and menus
- Task and action lists
- Status and notification feeds
- Search results and data tables

**Business Context:** Well-designed lists improve content scanability by 45% and reduce task completion time by 30%. Proper use of visual hierarchy and interactive states increases engagement rates by 25%. Lists with clear structure and consistent patterns reduce cognitive load and improve user satisfaction scores by 35%.

## Anatomy

The List component consists of the following parts:

1. **List Container** - Wrapper for all list items
2. **List Item** - Individual row containing content
3. **Leading Element** - Icon, avatar, or visual indicator (left side)
4. **Content Area** - Primary and secondary text
5. **Trailing Element** - Actions, indicators, or icons (right side)
6. **Section Header** - Optional label for grouped items
7. **Divider** - Visual separator between items or sections

## Usage Guidelines

### When to Use
- âœ… Displaying collections of similar items
- âœ… Navigation menus and option lists
- âœ… Settings and configuration screens
- âœ… Contact and user directories
- âœ… File browsers and document lists
- âœ… Search results and filtered data
- âœ… Activity feeds and notifications
- âœ… Multi-select interfaces

### When NOT to Use
- âŒ Tabular data with multiple columns (use Table)
- âŒ Highly structured grid layouts (use Grid or Card Grid)
- âŒ Minimal items (under 3) - use inline elements
- âŒ Complex hierarchical data (use Tree View)
- âŒ Horizontal scrolling content (use Carousel)
- âŒ Comparison of data points (use Table or Chart)

### Best Practices
- **Consistency:** Use same item structure throughout a list
- **Scanability:** Place most important info first (left-to-right)
- **Visual Hierarchy:** Use text size and weight to show importance
- **Icons:** Use consistent icon style and size (16px or 20px)
- **Touch Targets:** Minimum 44px height for interactive items
- **Loading States:** Show skeleton loaders for slow-loading lists
- **Empty States:** Provide helpful message when list is empty
- **Pagination:** Virtualize or paginate very long lists (over 100 items)

## Behavior / Interaction States

| State | Description | Design Tokens |
|-------|-------------|---------------|
| **Default** | List item in idle state | Background: `white`, Text: `--ds-color-text-primary` |
| **Hover** | Mouse over interactive item | Background: `--ds-color-bg-surface`, Cursor: `pointer` |
| **Focus** | Keyboard navigation highlight | Ring: `--ds-color-intent-primary`, Ring Width: `2px`, Offset: `2px` |
| **Active/Pressed** | Click or touch state | Background: `--ds-color-neutral-200` |
| **Selected** | Item in selected state | Background: `--ds-color-intent-primary` with 10% opacity |
| **Disabled** | Unavailable item | Opacity: `50%`, Cursor: `not-allowed` |
| **Read/Unread** | Different visual weights | Unread: bold text, Read: normal weight |
| **Error State** | Item with error or warning | Border: `--ds-color-intent-error`, Icon: error color |

### Interaction Details

**Non-Interactive List:**
- Items are not clickable
- Used for displaying static information
- No hover or focus states
- Good for displaying reference data

**Interactive List:**
- Items respond to clicks and keyboard
- Hover state provides visual feedback
- Focus follows keyboard navigation
- Can be single-select or multi-select

**Multi-Select Behavior:**
- Checkboxes or radio buttons in leading element
- Click anywhere on item to toggle selection
- Visual feedback shows selected state
- Support Shift+Click for range selection

**Keyboard Navigation:**
- `Tab`: Focus on list container
- `Arrow Down`: Navigate to next item
- `Arrow Up`: Navigate to previous item
- `Home`: Jump to first item
- `End`: Jump to last item
- `Enter`/`Space`: Activate focused item
- Type characters: Jump to item starting with typed letters

## Variants / Sizes / Densities

### By Style Variant

#### Default
**Visual:** Plain background, no border or shadow
**Use When:** Standard content, inline with page layout
**Properties:** `variant="default"`

#### Elevated
**Visual:** Shadow and border, raised appearance
**Use When:** Cards, panels, need visual separation
**Properties:** `variant="elevated"`

#### Outlined
**Visual:** Border only, no shadow
**Use When:** Grouped content, subtle separation
**Properties:** `variant="outlined"`

### By Density

#### Compact
**Visual:** Reduced vertical padding (8px)
**Use When:** Dense data displays, dashboards, small screens
**Properties:** `density="compact"`
**Item Height:** 48px minimum

#### Default
**Visual:** Standard padding (12px vertical)
**Use When:** Most use cases, balanced spacing
**Properties:** `density="default"` (or not specified)
**Item Height:** 56px minimum

#### Comfortable
**Visual:** Increased padding (16px vertical)
**Use When:** Touch interfaces, accessibility focus, relaxed layouts
**Properties:** `density="comfortable"`
**Item Height:** 72px minimum

### By Line Count

#### One-Line
**Visual:** Single line of text, icons align center
**Use When:** Simple labels, short names, navigation
**Min Height:** 48px default, 40px compact, 56px comfortable

#### Two-Line
**Visual:** Primary and secondary text lines
**Use When:** Items with supporting information
**Min Height:** 56px default, 48px compact, 64px comfortable

#### Three-Line
**Visual:** Primary, secondary, and supporting text
**Use When:** Rich content with multiple details
**Min Height:** 72px default, 64px compact, 88px comfortable

### By Content Type

#### Text Only
- Simple labels or descriptions
- No leading or trailing elements

#### With Icons
- Leading icon for visual identification
- Trailing icon for actions or status

#### With Avatars
- Leading circular avatar (32px, 40px, or 48px)
- User or entity identification

#### With Actions
- Trailing buttons or icon buttons
- Interactive elements for item actions

#### With Status
- Badges, chips, or indicators
- Show state, category, or count

## Accessibility

### Keyboard Navigation

| Key | Action |
|-----|--------|
| `Tab` | Move focus to list or first interactive element |
| `Arrow Down` | Move to next list item |
| `Arrow Up` | Move to previous list item |
| `Home` | Jump to first list item |
| `End` | Jump to last list item |
| `Enter` / `Space` | Activate focused item (if interactive) |
| `A-Z` | Type-ahead: focus item starting with typed character |

**Focus Management:**
- List container has `tabIndex={0}` if navigable
- Items have `tabIndex={0}` individually if separately focusable
- Focus visible on keyboard navigation
- Focus skips disabled items

### Screen Readers

**ARIA Attributes:**
```html
<!-- List Container -->
<div role="list">
  <!-- List Item -->
  <div role="listitem">
    <div>Item content</div>
  </div>

  <!-- Interactive Item -->
  <div role="listitem" tabIndex={0}>
    <button>Clickable item</button>
  </div>

  <!-- With Section Headers -->
  <div role="group" aria-labelledby="section-header-id">
    <div id="section-header-id" role="heading" aria-level={3}>
      Section Header
    </div>
    <div role="listitem">Item 1</div>
    <div role="listitem">Item 2</div>
  </div>
</div>
```

**Announcements:**
- List announces total item count on focus
- Each item announces its content when focused
- Selected state announced: "Selected, Item Name"
- Disabled state announced: "Disabled, Item Name"
- Multi-line items: primary text announced first

### Focus Management

- List receives single tab stop
- Arrow keys navigate between items (roving tabindex)
- Focus visible on all interactive elements
- Focus persists when list updates

### Color Contrast

- **Primary Text:** `--ds-color-text-primary` (#2B2D3F) - 4.5:1 minimum
- **Secondary Text:** `--ds-color-text-muted` (#70717D) - 4.5:1 minimum
- **Icons:** Inherit text color or use color tokens
- **Hover State:** 3:1 contrast against white
- **Selected State:** Maintains text contrast against background
- **Borders:** `--ds-color-border-muted` - 3:1 against backgrounds

### Additional Considerations

- Minimum touch target: 44px Ã— 44px
- Support browser zoom up to 200%
- Visible focus indicators on all items
- Don't rely solely on color for status
- Provide text alternatives for icon-only items
- Support screen reader mode for complex lists

## Content Guidelines

### Writing Style

âœ… **Do:**
- Use clear, descriptive primary text
- Keep secondary text concise (under 50 characters)
- Use sentence case for text
- Place most important info first
- Use consistent terminology
- Show timestamps in relative format ("2 hours ago")

âŒ **Don't:**
- Use all caps or Title Case
- Write long paragraphs in list items
- Use technical jargon without explanation
- Mix different content structures
- Truncate important information
- Use vague labels ("Item 1", "Option")

### Text Hierarchy

**Primary Text (One-Line):**
- Main item name or label
- Font: medium weight, 14px
- Color: `--ds-color-text-primary`

**Secondary Text (Two-Line):**
- Supporting information or description
- Font: regular weight, 12px
- Color: `--ds-color-text-muted`

**Supporting Text (Three-Line):**
- Additional context or details
- Font: regular weight, 12px
- Color: `--ds-color-text-muted`

## Responsive Behavior

### Desktop (1024px and up)
- Full width or constrained to content area
- Hover states fully functional
- Multi-column layouts possible
- Dense information density supported

### Tablet (768px - 1023px)
- Responsive width adapts to container
- Touch-optimized spacing (comfortable density)
- Hover states may not work (touch devices)
- Two-column layouts for wide lists

### Mobile (under 768px)
- Full-width items
- Increased touch targets (48px minimum)
- Comfortable or default density preferred
- Single column layout
- Consider infinite scroll for long lists
- Swipe actions for common operations

### Example 3: Interactive List with Selection

```tsx
import { useState } from 'react';
import {
  List,
  ListItem,
  ListItemContent,
  ListItemLeading,
  ListItemText,
  ListItemSecondaryText,
} from '@/components/ui/list';
import { Checkbox } from '@/components/ui/checkbox';

function SelectableList() {
  const [selectedIds, setSelectedIds] = useState<string[]>([]);

  const items = [
    { id: '1', title: 'Service Location 1', address: '123 Main St, Suite 100' },
    { id: '2', title: 'Service Location 2', address: '456 Business Ave' },
    { id: '3', title: 'Service Location 3', address: '789 Corporate Blvd' },
  ];

  const toggleSelection = (id: string) => {
    setSelectedIds(prev =>
      prev.includes(id)
        ? prev.filter(itemId => itemId !== id)
        : [...prev, id]
    );
  };

  return (
    <List variant="outlined" className="w-full max-w-md">
      {items.map(item => (
        <ListItem
          key={item.id}
          variant="clickable"
          selected={selectedIds.includes(item.id)}
          onClick={() => toggleSelection(item.id)}
        >
          <ListItemLeading>
            <Checkbox
              checked={selectedIds.includes(item.id)}
              onChange={() => toggleSelection(item.id)}
            />
          </ListItemLeading>
          <ListItemContent lines="two">
            <ListItemText>{item.title}</ListItemText>
            <ListItemSecondaryText>{item.address}</ListItemSecondaryText>
          </ListItemContent>
        </ListItem>
      ))}
    </List>
  );
}
```

```tsx
            <Badge variant={item.statusVariant} className={item.statusClass}>
              {item.status}
            </Badge>
          </ListItemTrailing>
        </ListItem>
      ))}
    </List>
  );
}
```
