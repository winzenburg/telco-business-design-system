## üé® Design System Showcase

Each workflow demonstrates our design system's core strengths:

### Token-First Architecture
```tsx
// ‚úÖ All styles use design tokens
<div className="text-[var(--ds-color-text-primary)]">
  <Input error={!!errors.email} />
  <p className="text-[var(--ds-color-intent-destructive)]">
    {errors.email}
  </p>
</div>
```

### Border Hierarchy (ENFORCED)
- **Form inputs**: `border-neutral-400` (#B4B5BB) - darker borders for interaction
- **Structural elements**: `border-neutral-300` (#DDDDE2) - lighter borders for organization
- **Error states**: `border-destructive` - red borders for validation errors

### 5-State Form Model
Every form component supports:
1. **Default** - Neutral appearance, ready for input
2. **Focused** - Primary-500 border and focus ring
3. **Filled** - Content present, normal styling
4. **Error** - Destructive border and messaging
5. **Disabled** - Reduced opacity, not-allowed cursor

## üí° Usage Tips

### Copy-Paste Ready
All workflows are **production-ready**. You can copy the entire component and adapt it to your needs.

### Validation Patterns
Each workflow demonstrates different validation strategies:
- **Real-time**: Validate on every keystroke (CVV, phone)
- **On blur**: Validate when field loses focus (email)
- **On submit**: Validate all fields when form submitted

### State Management
Workflows show how to manage complex state:
```tsx
const [formData, setFormData] = useState({ /* ... */ });
const [errors, setErrors] = useState<FormErrors>({});
const [isSubmitting, setIsSubmitting] = useState(false);

const handleChange = (field: string, value: any) => {
  setFormData(prev => ({ ...prev, [field]: value }));
  // Clear error when user starts typing
  if (errors[field]) {
    setErrors(prev => ({ ...prev, [field]: undefined }));
  }
};
```

### Responsive Patterns
```tsx
// Desktop: 3-column grid
// Tablet: 2-column grid
// Mobile: 1-column stack
<div className="grid grid-cols-1 lg:grid-cols-3 gap-6">
  <div className="lg:col-span-2">{/* Main content */}</div>
  <div>{/* Sidebar */}</div>
</div>
```
