## Design Principles & Technical Approach

Our design system is built on five core principles that ensure consistency, accessibility, and maintainability:

### 1. **Token-First Architecture**
Every color, spacing, typography value flows from design tokens. No hard-coded hex values, no magic numbers‚Äîjust semantic tokens that adapt to themes and contexts.

### 2. **Border Hierarchy System**
- **Form inputs** (input, textarea, select, checkbox): Use `neutral-400` (#B4B5BB)
- **Structural elements** (table, card, accordion, dialog): Use `neutral-300` (#DDDDE2)
- This creates visual hierarchy that guides user attention to interactive elements

### 3. **Accessibility by Default**
- **WCAG 2.1 AA compliant** - All components tested for color contrast, keyboard navigation
- **Full keyboard operability** - Tab navigation, arrow keys, escape to close
- **Screen reader support** - Proper ARIA labels, roles, and live regions
- **Axe-core tested** - Zero violations in all components

### 4. **Performance-Optimized**
- **Tree-shakeable** - Import only what you use
- **Lazy-loaded** - Heavy components load on demand
- **SSR-safe** - Server-side rendering compatible
- **Zero layout thrash** - Theme switching doesn't cause reflows

### 5. **ShadCN/UI-First**
Never build from scratch‚Äîstart with proven ShadCN/UI implementations, then enhance with our design tokens and business logic. This prevents reinventing the wheel and ensures best practices.

## Featured Enterprise Patterns

### üì± ResponsiveLayout
Adaptive navigation that automatically switches between desktop sidebar and mobile menu based on viewport width. Supports flexible breakpoints (sm/md/lg), custom menu items, and user profile sections.

**Use Cases**: Admin dashboards, SaaS applications, internal tools

### üîê PermissionGate
Role-based access control built directly into components. Restrict entire sections, individual actions, or show degraded states based on user permissions.

**Use Cases**: Multi-tenant applications, role-based dashboards, enterprise admin panels

### üìä SummaryCard
Display key metrics with status indicators (success, warning, error, info). Perfect for dashboards and monitoring interfaces.

**Use Cases**: System health dashboards, billing summaries, network status

### ü™ú Stepper / GuidedFlows
Multi-step workflows with progress tracking, validation, and navigation controls. Keeps users oriented in complex processes.

**Use Cases**: Onboarding flows, checkout processes, configuration wizards, multi-step forms

## Developer Experience

### Auto-Validated by Claude Code
Every component passes automated checks for:
- Border hierarchy compliance (form inputs vs structural elements)
- Token usage (no hard-coded colors or spacing)
- Accessibility (keyboard navigation, ARIA attributes, color contrast)
- TypeScript types (full type safety)

### Pre-commit Quality Gates
Automatic linting, type-checking, and test execution before every commit prevents regressions:

```bash
npm run validate:tokens      # No hard-coded values
npm run validate:a11y        # Accessibility audit
npm run test                 # Unit + integration tests
npm run test:vrt             # Visual regression tests
```

### Visual Regression Testing
Playwright snapshots prevent visual regressions. Every component has VRT coverage with diff thresholds ‚â§1%.

### Comprehensive Documentation

- **Anatomy** - Visual breakdown of component parts

- **Variants** - All size/type/state combinations
- **Usage Guidelines** - When to use, when NOT to use

## Get Started

### Installation

```bash
# Install the design system
npm install @comcast-business/design-system

# Install peer dependencies
npm install react react-dom tailwindcss
```

### Import Components

```tsx
import { Button, Input, Card } from '@comcast-business/design-system';

function App() {
  return (
    <Card>
      <Input placeholder="Enter email" />
      <Button variant="primary">Submit</Button>
    </Card>
  );
}
```

### Use Design Tokens

```css
/* Import CSS variables */
@import '@comcast-business/tokens/css/variables.css';

/* Use tokens in your styles */
.custom-element {
  color: var(--ds-color-primary-500);
  padding: var(--ds-spacing-4);
  border-radius: var(--ds-radius-md);
}
```
