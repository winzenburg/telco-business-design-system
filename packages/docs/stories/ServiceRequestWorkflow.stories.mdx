# Service Request Workflow

## Design Intent

The Service Request Workflow guides users through the complete process of requesting new business services, from initial requirements gathering through approval and provisioning. This workflow pattern supports:

- **Multi-step forms**: Progressive disclosure for complex service configurations
- **Validation flows**: Real-time validation and error handling at each step
- **Approval workflows**: Request routing, stakeholder reviews, and authorization chains
- **Progress tracking**: Clear status indicators and timeline visibility
- **Documentation management**: Terms, contracts, and service level agreements

### When to Use

- New service provisioning requests (Internet, Phone, TV, Security)
- Service upgrade workflows and plan changes
- Additional location service requests
- Enterprise onboarding and account setup
- Custom service configuration requests

### When to Avoid

- Emergency service requests (use Service Management workflow)
- Billing inquiries and payment processing (use Account Management patterns)
- Technical support requests (use Support Portal patterns)
- Simple account changes (use Settings patterns)

## Accessibility Contract

### Keyboard Interactions

| Key | Action |
|-----|--------|
| `Tab` | Navigate between form fields and interactive elements |
| `Shift + Tab` | Navigate backwards through form fields |
| `Enter` | Submit form sections, activate buttons |
| `Space` | Toggle checkboxes and radio buttons |
| `Arrow Keys` | Navigate between radio button options |
| `Escape` | Cancel form submission, close modals |
| `F6` | Navigate between main form sections |

### ARIA Patterns

- Form sections use proper `fieldset` and `legend` elements
- Multi-step progress uses `role="tablist"` with step indicators
- Form validation errors use `role="alert"` for screen reader announcements
- Required fields include `aria-required="true"` attribute
- Form sections include `aria-describedby` for help text
- Progress indicators include `aria-current="step"` for active step

### Focus Management

- Clear visual focus indicators on all form fields
- Focus automatically moves to first field in each step
- Error fields receive focus when validation fails
- Step completion maintains focus context appropriately
- Modal dialogs trap focus within the dialog content

## Responsive/Adaptive Behavior

### Breakpoints

- **Mobile (320px+)**: Single column layout, collapsible sections, simplified step indicators
- **Tablet (768px+)**: Two-column layout for shorter forms, horizontal step progress
- **Desktop (1024px+)**: Multi-column layout with sidebar progress, inline validation
- **Large Desktop (1440px+)**: Wide layout with expanded help documentation panel

### Form Adaptations

- Mobile: Full-width fields, vertical button stacking, drawer-style step navigation
- Tablet: Grouped field layouts, horizontal button arrangements
- Desktop: Multi-column field layouts, inline help text, persistent navigation
- Large: Side-by-side form and preview, expanded documentation

## Performance Notes

### SSR-Safe Implementation

- Form state hydrates progressively with validation rules
- Step progress renders server-side with skeleton placeholders
- File upload components load on-demand with proper fallbacks
- Form validation works without JavaScript (progressive enhancement)

### Bundle Impact

- Base workflow: ~6kb gzipped (excluding validation library)
- Multi-step navigation: ~2kb for step management
- File upload handling: ~4kb loaded on-demand
- Form validation: Uses existing design system validation (~1kb additional)

### Form Performance

- Field validation debounced to 300ms to prevent excessive API calls
- Large form sections lazy-load to improve initial page load
- File uploads use chunked uploads with progress indicators
- Form auto-save prevents data loss during long completion times

      const newErrors: Record<string, string> = {};

      switch (step) {

          if (!formData.companyName.trim()) {
            newErrors.companyName = 'Company name is required';

          if (!formData.contactName.trim()) {
            newErrors.contactName = 'Contact name is required';

            newErrors.email = 'Email address is required';
          } else if (!formData.email.includes('@')) {
            newErrors.email = 'Please enter a valid email address';

            newErrors.phone = 'Phone number is required';

            newErrors.address = 'Street address is required';

            newErrors.city = 'City is required';

          if (!formData.state.trim()) {
            newErrors.state = 'State is required';
          }
          if (!formData.zip.trim()) {
            newErrors.zip = 'ZIP code is required';
          }
          break;

          if (formData.services.length === 0) {
            newErrors.services = 'Please select at least one service';
          }
          if (!formData.businessType) {
            newErrors.businessType = 'Please select a business type';
          }
          break;

          if (!formData.terms) {
            newErrors.terms = 'You must accept the terms and conditions';
          }
          break;

      return Object.keys(newErrors).length === 0;

      if (validateStep(currentStep)) {
        setCurrentStep(prev => Math.min(prev + 1, totalSteps));

        services: checked

                        <div className="flex items-center space-x-2">
                          <Checkbox
                            id="sip"
                            checked={formData.services.includes('sip')}
                            onCheckedChange={(checked) => 
                              handleServiceChange('sip', !!checked)
                            }
                          />
                          <Label htmlFor="sip" className="cursor-pointer">
                            SIP Trunking
                          </Label>
                        </div>
                      </div>
                    </div>

                    <div>
                      <h4 className="font-medium text-primary mb-4">
                        Business Type <span className="text-destructive-500">*</span>
                      </h4>
                      <RadioGroup
                        value={formData.businessType}
                        onValueChange={(value) => setFormData(prev => ({
                          ...prev, 
                          businessType: value
                        }))}
                        aria-required="true"
                        aria-invalid={!!errors.businessType}
                      >
                        <div className="flex items-center space-x-2">
                          <RadioGroupItem value="small" id="small" />
                          <Label htmlFor="small" className="cursor-pointer">
                            Small Business (1-20 employees)
                          </Label>
                        </div>
                        <div className="flex items-center space-x-2">
                          <RadioGroupItem value="medium" id="medium" />
                          <Label htmlFor="medium" className="cursor-pointer">
                            Medium Business (21-100 employees)
                          </Label>
                        </div>
                        <div className="flex items-center space-x-2">
                          <RadioGroupItem value="enterprise" id="enterprise" />
                          <Label htmlFor="enterprise" className="cursor-pointer">
                            Enterprise (100+ employees)
                          </Label>
                        </div>
                      </RadioGroup>
                      {errors.businessType && (
                        <p role="alert" className="text-sm text-destructive-600 mt-2">
                          {errors.businessType}
                        </p>
                      )}
                    </div>
                  </CardContent>
                </Card>
              )}

              {/* Step 4: Review & Submit */}
              {currentStep === 4 && (
                <div className="space-y-6">
                  <Card>
                    <CardHeader>
                      <CardTitle>Review Your Request</CardTitle>
                      <CardDescription>
                        Please review your service request details
                      </CardDescription>
                    </CardHeader>
                    <CardContent className="space-y-6">
                      <div>
                        <h4 className="font-medium text-primary mb-3">Company Information</h4>
                        <div className="bg-surface rounded-lg p-4 space-y-2">
                          <p><strong>Company:</strong> {formData.companyName}</p>
                          <p><strong>Contact:</strong> {formData.contactName}</p>
                          <p><strong>Email:</strong> {formData.email}</p>
                          <p><strong>Phone:</strong> {formData.phone}</p>
                        </div>
                      </div>

                      <div>
                        <h4 className="font-medium text-primary mb-3">Service Address</h4>
                        <div className="bg-surface rounded-lg p-4">
                          <p>
                            {formData.address}<br />
                            {formData.city}, {formData.state} {formData.zip}
                          </p>
                        </div>
                      </div>

                      <div>
                        <h4 className="font-medium text-primary mb-3">Selected Services</h4>
                        <div className="bg-surface rounded-lg p-4 space-y-2">
                          {formData.services.length > 0 ? (
                            formData.services.map(service => (
                              <Badge key={service} variant="outline">
                                {service.replace('-', ' ').replace(/\b\w/g, l => l.toUpperCase())}
                              </Badge>
                            ))
                          ) : (
                            <p className="text-muted">No services selected</p>
                          )}
                          <p><strong>Business Type:</strong> {formData.businessType}</p>
                        </div>
                      </div>

                      <Separator />

                      <div className="space-y-4">
                        <div className="flex items-start space-x-2">
                          <Checkbox
                            id="terms"
                            checked={formData.terms}
                            onCheckedChange={(checked) => setFormData(prev => ({
                              ...prev, 
                              terms: !!checked
                            }))}
                            aria-required="true"
                            aria-invalid={!!errors.terms}
                            aria-describedby={errors.terms ? "terms-error" : undefined}
                          />
                          <Label htmlFor="terms" className="cursor-pointer text-sm">
                            I agree to the Terms and Conditions and Service Level Agreement{' '}
                            <span className="text-destructive-500">*</span>
                          </Label>
                        </div>
                        {errors.terms && (
                          <p id="terms-error" role="alert" className="text-sm text-destructive-600">
                            {errors.terms}
                          </p>
                        )}

                        <Alert className="border-primary-200 bg-primary-50">
                          <AlertDescription className="text-primary-800">
                            <strong>Next Steps:</strong> After submission, a business specialist will review your request and contact you within 1-2 business days to discuss implementation details and pricing.
                          </AlertDescription>
                        </Alert>
                      </div>
                    </CardContent>
                  </Card>
                </div>
              )}

              {/* Navigation Buttons using design tokens */}
              <div className="flex items-center justify-between pt-6">
                <Button
                  variant="outline"
                  onClick={handlePrevious}
                  disabled={currentStep === 1}
                >
                  Previous
                </Button>

                <div className="flex items-center gap-3">
                  {currentStep < totalSteps ? (
                    <Button onClick={handleNext}>
                      Continue
                    </Button>
                  ) : (
                    <>
                      <Button variant="outline">
                        Save Draft
                      </Button>
                      <Button
                        onClick={() => {
                          if (validateStep(currentStep)) {
                            // Handle form submission
                            alert('Service request submitted successfully!');
                          }
                        }}
                        disabled={!formData.terms}
                      >
                        Submit Request
                      </Button>
                    </>
                  )}
                </div>
              </div>
            </div>
          </div>
        </div>
      </div>
    );
  },
};

// Accessibility Testing Story
export const AccessibilityTest: Story = {
  render: Default.render,
  play: async ({ canvasElement }) => {
    const canvas = within(canvasElement);
    
    // Test for accessibility violations
    const results = await axe(canvasElement);
    expect(results).toHaveNoViolations();
    
    // Test form field accessibility
    const companyNameField = canvas.getByLabelText(/company name/i);
    expect(companyNameField).toHaveAttribute('aria-required', 'true');
    
    // Test step navigation accessibility
    const steps = canvas.getAllByText(/step \d of \d/i);
    expect(steps.length).toBeGreaterThan(0);
    
    // Test form validation
    const continueButton = canvas.getByRole('button', { name: /continue/i });
    fireEvent.click(continueButton);
    
    // Check that validation errors are announced to screen readers
    const errorMessages = canvas.getAllByRole('alert');
    expect(errorMessages.length).toBeGreaterThan(0);
  },
};
